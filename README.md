ServiceNow Client Script & UI Policy Project

Project Overview
This project demonstrates how Client Scripts and UI Policies are used in ServiceNow to control form behavior dynamically.

Objective
To automate form behavior using:
- Client Scripts
- UI Policies

 Features
- Fields become mandatory dynamically
- Fields are shown/hidden based on conditions
- Improved user experience on forms

 Use Case
When user selects a specific value:
- Some fields become mandatory
- Some fields become read-only
- Some fields become visible

 Files
- Client_Script_UI_Policy_Update_Set.xml

How to Use
1. Import the XML into ServiceNow
2. Preview update set
3. Commit update set
4. Test on form
